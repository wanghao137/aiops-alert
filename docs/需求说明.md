# 智能监控告警系统 · 需求说明

## 1. 背景与定位

随着企业数字化深入，IT 基础设施和数据作业链路越来越复杂，传统告警系统普遍存在三个问题：

1. **配置繁琐**：规则要填几十个字段，运维要懂监控语言才能上手
2. **告警刷屏**：同一故障刷出几十条告警，无法判断主因
3. **缺乏指引**：告警来了只看到"CPU 92%"，不知道为什么、影响了什么、该怎么办

本系统在传统告警管理（对象、规则、渠道、事件、看板）基础上，引入 AI 能力，重塑这三个体验。

## 2. 用户角色

| 角色 | 关注点 |
|---|---|
| 运维工程师 | 快速建规则、第一时间响应告警、知道怎么处理 |
| 团队 Leader | 看团队告警全貌、评估健康度、分析高频问题 |
| 系统管理员 | 维护对象/渠道、配置 AI 模型、管理权限 |

## 3. 功能模块

### 3.1 监控对象管理
- 4 类对象：服务器 / 数据库 / 数据同步作业 / 数据加工作业
- CRUD + 启停 + 按类型/关键字筛选
- 扩展属性按对象类型差异化（IP/端口/作业编码等）

### 3.2 告警规则管理 ⭐ AI 增强
- 传统：选对象 → 选指标 → 设阈值 → 选渠道 → 保存
- **AI 自然语言建规则**：用一句中文描述，自动生成结构化规则
- **AI 阈值推荐**：基于历史指标分位数，给出高/中/低三档建议
- 多条件 AND/OR、连续触发次数、时间窗、最小间隔、重复/恢复通知

### 3.3 告警渠道管理
- 3 类渠道：企业微信 / 邮件 / 短信
- CRUD + 测试发送
- 渠道连通性实时反馈（最近一次发送状态）

### 3.4 告警事件中心 ⭐ AI 增强
- 全生命周期：待处理 / 已确认 / 已恢复 / 已关闭
- 处理动作：确认 / 恢复 / 关闭 / 备注（全部留痕）
- 通知发送记录 + 失败重发
- **AI 摘要卡**：每条事件自动生成「发生了什么 / 影响范围 / 可能原因 / 建议动作」
- **Incident 视图**：同对象短时间多告警自动合并

### 3.5 总览大屏
- 实时核心指标（事件总数 / 待处理 / 严重紧急 / 通知失败）
- 7 天告警趋势图
- 状态/级别/对象类型分布
- 规则命中 Top
- 实时告警流（SSE 推送）
- AI 生成"今日态势简报"

### 3.6 命令面板（全局）⭐
- 快捷键 `Cmd+K` / `Ctrl+K` 唤起
- 自然语言查询："现在哪些对象在告警？""把 prod-redis-01 的所有告警关闭"
- LLM 解析后调用内部 API，返回结构化结果

## 4. 非功能需求

| 维度 | 要求 |
|---|---|
| 响应性 | 列表页首屏 < 1s；告警从触发到前端可见 < 3s |
| 实时性 | 新告警通过 SSE 推送，无需刷新 |
| 可观测 | LLM 调用全部留痕（耗时、token、成本） |
| 美观度 | 深色主题为主、动效自然、数字等宽 |
| 可演示 | 一键初始化样例数据、一键模拟告警触发 |

## 5. 演示路径

5-10 分钟评委视角：

1. **总览大屏** — 实时数字跳动，新告警从右侧滑入
2. **AI 建规则** — 一句话生成规则，秒级填表
3. **触发演示告警** — toast 弹出 → 详情页有 AI 摘要
4. **Incident 视图** — 多告警自动归并展示
5. **Cmd+K 询问** — 自然语言查询当前告警态势

## 6. 范围边界

**做**
- 告警管理 + AI 增强 + 实时推送 + 美观 UI

**不做**
- 完整时序数据库与采集代理（用模拟器生成指标）
- 分布式调度引擎
- 智能根因分析图谱（v2 再考虑）
- 电话语音告警
