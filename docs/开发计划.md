## 后续开发计划

按 **必做 → 加分 → 锦上添花** 三档优先级。

### 🔴 P0 必做（演示前必须完成）

#### 1. 真实告警引擎"故事模式"（半天）
**问题**：当前模拟器 ±5% 随机游走，规则的"连续触发 3 次"很难命中，演示时不一定能"自动触发"出告警。

**方案**：
- 模拟器增加"故事模式"：每 3-5 分钟轮流让某个对象进入异常状态（比如 prod-mysql 主从延迟开始爬升）
- 触发后能看到从模拟器写指标 → 引擎评估 → 命中 → 写事件 → AI 摘要 → SSE 推送的完整自动闭环
- 演示时不靠手动触发，让评委看到自动触发的瞬间

#### 2. SSE 长链接稳健化（1 小时）
**现象**：前端日志偶尔 `ECONNRESET`，反向代理或浏览器空闲会断开。

**方案**：每 20s 推一条 ping 事件 + 前端自动重连。

#### 3. 演示数据完善（1 小时）
- demo seed 加 6-8 条历史 7 天的事件分布到不同日期，让趋势图有曲线
- 给一两条历史事件预生成 AI 摘要，避免演示时全靠现场调用

#### 4. 异常处理 + 加载态打磨（半天）
- 详情页 AI 摘要 PENDING 时图标动画
- 各页 loading 骨架屏统一
- 网络断时全局 banner 提示

---

### 🟡 P1 加分（直接拿分点）

#### 5. 阈值推荐升级 → AI 解释（1 天）
当前阈值推荐是纯历史分位数。可以让 LLM 看完分位数 + 业务场景后给"为什么推荐这个值"的解释。

#### 6. AI 调用统计页（半天）
利用现有 `ai_call_log` 表数据，做"AI 工程"页：
- 今日调用次数 / token 消耗 / 成功率
- 按场景分类（NL2RULE / EVENT_SUMMARY / CHAT / THRESHOLD）
- 慢调用排行
- 估算成本（按模型每千 token 单价）

让评委看到"AI 用得多而且可观测"。

#### 7. 告警规则版本与一键回滚（1 天）
- 规则每次保存时存一份快照到 alert_rule_version 表
- 详情页加"版本历史" tab，可对比 diff
- 一键回滚

#### 8. 命令面板增强（1 天）
扩展 4 类操作：
- "把 prod-redis-01 的所有告警关闭" → 真的批量关
- "停用规则 RULE-DB-LAG" → 真的停
- "为 prod-mysql 创建一条 CPU 告警" → 直接进 NL2Rule
- "今天最高的告警是什么" → 流式回答

### 🟢 P2 锦上添花（有时间一定做）

#### 9. 每日态势简报（半天）
- 定时任务（每天 8:00）调 LLM

- 推送到企微 / 邮件
- 总览大屏顶部加"今日简报"卡片

#### 10. 暗黑 / 明亮主题切换（半天）
现在锁深色，加个明亮主题切换。

#### 11. 大屏全屏模式（半天）
F11 进全屏，隐藏侧栏 + 顶栏 + 30s 轮播刷新，NOC 大屏的样子。

#### 12. 告警关联分析（1.5 天）
基于 Incident 的事件，让 LLM 输出"根因路径分析"：
- "prod-redis 内存满 → 5 分钟后 prod-web-01 响应延迟 → 8 分钟后 OS 慢查询激增"
- 时间线 + 因果箭头

#### 13. 演示 PPT + 视频（1 天）
- 5 分钟讲解视频（带字幕）
- 演示流程脚本
- 1-2 个备用演示路径，防止现场网络 / LLM 抖动

## 推荐冲刺顺序

按"价值 / 时间"比：

**第一周**：1 → 2 → 3 → 4 → 6 → 5（核心稳定 + AI 可观测）

**第二周**：8 → 9 → 12（命令面板增强 + 态势简报 + 根因分析）

**比赛前 3 天**：10 → 11 → 13（视觉惊艳 + 演示打磨）

**第三周以上**：7（规则版本，工程性强但 AI 含量低，看时间）

## 演示路径建议（5-10 分钟版）

1. **总览大屏** — 实时数字 + 趋势曲线，第一印象
2. **AI 一句话建规则**（30 秒）— 核心卖点，输入"当生产 MySQL 主从延迟超过 5 分钟，连续触发 3 次，发企微给 DBA 群，紧急级别"，AI 自动选对象、指标、阈值、级别、渠道
3. **触发演示告警** — 右上角 toast 滑入，详情页 AI 摘要四段（发生了什么 / 影响范围 / 可能原因 / 建议动作）流式生成
4. **Incident 视图** — 同对象多告警自动归并的故障组，时间轴展示
5. **命令面板 ⌘K** — "现在哪些对象在告警？"自然语言查询返回结构化结果
6. **AI 调用统计页**（如已完成）— 收尾，证明 AI 不是噱头是工程化能力
