## 后续开发计划

按 **必做 → 加分 → 锦上添花** 三档优先级。

### 🔴 P0 必做（演示前必须完成）

#### 1. 真实告警引擎"故事模式"（半天） ✅ 已完成
**问题**：当前模拟器 ±5% 随机游走，规则的"连续触发 3 次"很难命中。
**已实现**：`MetricSimulator.maybeStartStory()` 自动随机让对象进入异常 + `forceStory(objectId, metric)` 手动触发；前端 EventsView "启动故事模式"按钮接入。

#### 2. SSE 长链接稳健化（1 小时） ✅ 已完成
**已实现**：后端 `AlertStreamService.heartbeat()` 每 20s 发 ping；前端 `useSse` 60s watchdog 自动重连；新增全局 `NetworkBanner` 在 SSE 断开 ≥30s 或 5min 内 5xx ≥3 次时显示警示。

#### 3. 演示数据完善（1 小时） ✅ 已完成
**已实现**：`DemoDataService.seedHistorical()` 在 seed 时回填 12 条历史 AlertEvent 分散到近 7 天 + 3 条预生成 SUCCESS 状态 AI 摘要 + 每个 (object, metric) 维度 ~98 条 7 天 MetricSample（让阈值推荐返回 source=HISTORY）。

#### 4. 异常处理 + 加载态打磨（半天） ✅ 已完成
**已实现**：AiSummaryCard PENDING 状态终端打字机动画；5 个 ListView 接入统一 SkeletonList；ErrorPage 404 + 500 兜底；NetworkBanner 网络异常提示。

### 🟡 P1 加分（直接拿分点）

#### 5. 阈值推荐升级 → AI 解释（1 天）
当前阈值推荐是纯历史分位数。可以让 LLM 看完分位数 + 业务场景后给"为什么推荐这个值"的解释。

#### 6. AI 调用统计页（半天） ✅ 已完成
利用 `ai_call_log` 表数据，做"AI 工程"页：
- 今日调用次数 / token 消耗 / 成功率 + 与昨日环比
- 按场景（NL2RULE / EVENT_SUMMARY / CHAT / THRESHOLD / OTHER）分布的环形图
- 7 日调用趋势折线
- 慢调用 Top 10（行内展开 prompt + response）
- 估算成本卡（基于 `llm_model_config.prompt_price_per_1k` / `completion_price_per_1k`）
- 流水分页 + 详情抽屉

**入口**：侧栏菜单"AI 调用统计"（`/ai-stats`）。后端 `AiCallStatsService` + `AiCallStatsController` 4 个端点 (`/overview` `/slow` `/logs` `/logs/{id}`)。

#### 7. 告警规则版本与一键回滚（1 天）
- 规则每次保存时存一份快照到 alert_rule_version 表
- 详情页加"版本历史" tab，可对比 diff
- 一键回滚

#### 8. 命令面板增强（1 天）
扩展 4 类操作：
- "把 prod-redis-01 的所有告警关闭" → 真的批量关
- "停用规则 RULE-DB-LAG" → 真的停
- "为 prod-mysql 创建一条 CPU 告警" → 直接进 NL2Rule
- "今天最高的告警是什么" → 流式回答

## 推荐冲刺顺序

按"价值 / 时间"比：

**第一周**：1 → 2 → 3 → 4 → 6 → 5（核心稳定 + AI 可观测）

**第二周**：8 → 9 → 12（命令面板增强 + 态势简报 + 根因分析）

## 演示路径建议（5-10 分钟版）

1. **总览大屏** — 实时数字 + 趋势曲线，第一印象
2. **AI 一句话建规则**（30 秒）— 核心卖点，输入"当生产 MySQL 主从延迟超过 5 分钟，连续触发 3 次，发企微给 DBA 群，紧急级别"，AI 自动选对象、指标、阈值、级别、渠道
3. **触发演示告警** — 右上角 toast 滑入，详情页 AI 摘要四段（发生了什么 / 影响范围 / 可能原因 / 建议动作）流式生成
4. **Incident 视图** — 同对象多告警自动归并的故障组，时间轴展示
5. **命令面板 ⌘K** — "现在哪些对象在告警？"自然语言查询返回结构化结果
6. **AI 调用统计页**（如已完成）— 收尾，证明 AI 不是噱头是工程化能力
