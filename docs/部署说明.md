## 2. 三个长跑进程

| terminalId | 用途 | 启动命令 | 工作目录 |
|---|---|---|---|
| 2 | 后端 | `mvnw spring-boot:run -q`（JAVA_HOME 指 JDK17） | `aiops-alert/backend` |
| 8 | cloudflared 隧道 | `cloudflared tunnel --url http://localhost:4173 --no-autoupdate --logfile %TEMP%\cf-tunnel.log` | `monitoring-alert` |
| 12 | vite preview | `npm run preview` | `aiops-alert/frontend` |

## 3. 关键配置

### `frontend/vite.config.ts` — preview 块
```ts
preview: {
  port: 4173,
  host: '0.0.0.0',
  allowedHosts: true,           // 接受任意 host header（cloudflared 子域是动态的）
  proxy: {
    '/api': {
      target: 'http://localhost:8090',
      changeOrigin: true
    }
  }
}
```

## 5. 排错 / 踩过的坑

### 坑 1：vite preview 返回 403 "Blocked request. This host"
- vite 5 默认只接受 localhost 和配置中列出的 host
- 修：`preview.allowedHosts: true`（或写成 `string[]` 列表）

### 坑 2：端口 4173 被别的项目占用
- 现象：preview 看似启动了，但流量落到陌生进程
- 排查：
  ```powershell
  Get-NetTCPConnection -LocalPort 4173 | Select OwningProcess
  Get-CimInstance Win32_Process -Filter "ProcessId=<pid>" | Select CommandLine
  ```
- 修：`Stop-Process -Id <pid> -Force` 然后重启 preview

### 坑 3：PowerShell 字符串 `&` 字面量
- `&` 在 PowerShell 里是命令调用符，单引号都不能转义
- 解法：`$AMP=[char]38` 然后 `"$AMP$AMP"`，或者用 cmd /c 包一层

### 坑 4：写中文文件
- `Set-Content` 默认 GBK，中文会乱码
- 解法：用 fs_write 工具，或 `[System.IO.File]::WriteAllText($path, $content, [System.Text.UTF8Encoding]::new($false))`

### 坑 5：JDK 版本
- 后端要 17，系统默认可能是 1.8
- 解法：每次启动设 JAVA_HOME 指向 `C:\Program Files\Eclipse Adoptium\jdk-17.0.19.10-hotspot`

## 6. 持久化部署方案（待做）

### 方案 A：Cloudflare 命名隧道（推荐）
- `cloudflared tunnel login`（浏览器登录）
- `cloudflared tunnel create aiops-alert`
- 在 Cloudflare 控制台绑定自有域名（CNAME）
- 写 `~/.cloudflared/config.yml`：
  ```yaml
  tunnel: <tunnel-id>
  credentials-file: <path>
  ingress:
    - hostname: aiops.yourdomain.com
      service: http://localhost:4173
    - service: http_status:404
  ```
- `cloudflared service install` 注册 Windows 服务，开机自启
- URL 永久不变，免费

### 方案 B：nginx + docker-compose（生产级）
- nginx 监听 80/443，托管 dist/ 静态 + 反代 /api
- docker-compose 把 backend / nginx / mysql 一锅起
- 自带 SSL（Let's Encrypt 或 Cloudflare）
- 文件位置：项目根 `docker-compose.yml` + `deploy/nginx.conf`

### 方案 C：systemd / Windows 服务
- 后端用 `nssm` 或 `winsw` 包成 Windows 服务
- nginx-portable 同样包成服务
- 适合单机长期跑，无 docker
