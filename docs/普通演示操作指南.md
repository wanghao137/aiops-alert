# AIOps Alert 普通演示操作指南

> 给非研发同事看的短版说明。目标是能启动、能演示、能知道出问题先看哪里。
>
> 最后更新：2026-05-27

## 1. 一句话介绍

AIOps Alert 是一个智能监控告警演示系统。它把“发现异常、生成告警、AI 总结原因、通知相关人、跟踪处理、复盘统计”串成一个闭环。

## 2. 怎么启动

1. 打开项目文件夹 `aiops-alert`。
2. 双击 `start.cmd`。
3. 等两个黑色窗口都启动完成：
   - `AIOps Backend (8090)`：后端服务。
   - `AIOps Frontend (5173)`：前端页面。
4. 浏览器会自动打开 `http://localhost:5173`。

如果浏览器没有自动打开，手动访问：`http://localhost:5173`。

停止系统：双击 `stop.cmd`，或者关闭两个黑色窗口。

## 3. 演示时怎么讲

建议按这 5 步讲，5 到 8 分钟即可。

| 步骤 | 页面 | 讲什么 |
|---|---|---|
| 1 | 总览大屏 | 当前有多少告警、趋势怎么样、AI 每日简报怎么看。 |
| 2 | 告警事件 | 点开一条告警，看 AI 总结“发生了什么、影响范围、可能原因、建议动作”。 |
| 3 | 触发演示 | 在告警事件页点“触发演示”，现场生成一条告警，看 AI 摘要从“思考中”变成“已生成”。 |
| 4 | 告警规则 | 用自然语言说一句需求，让 AI 生成告警规则。 |
| 5 | AI 调用统计 | 看今天调用了多少次 AI、成功率、慢调用和失败原因。 |

## 4. 重点页面怎么用

### 总览大屏

- 大数字表示当前待处理告警数量。
- “AI Brief” 是系统自动生成的每日简报。
- 右上角“大屏模式”适合投屏演示，按 `ESC` 退出。

### 告警事件

- 左边按状态筛选，右边看详情。
- AI 摘要有三种状态：
  - “思考中”：AI 正在生成。
  - “已生成”：可以看到原因、影响和建议。
  - “失败”：可以点重新生成。
- 通知日志里如果显示“演示成功 · 未真实发送”，说明这是演示模式，不会真的发企业微信、邮件或短信。

### 告警规则

- 普通新建：点“新增规则”，按表单填写。
- AI 新建：在 AI Builder 里输入一句话，例如：

```text
我想监控生产 MySQL 主库的主从延迟，超过 60 秒就发企微告警
```

系统会生成规则草稿，确认后可应用。

### 系统设置

这里配置大模型。配置好以后，事件摘要、规则生成、每日简报都会走默认模型。

DeepSeek 推荐先用：

| 字段 | 推荐值 |
|---|---|
| 提供商 | `DEEPSEEK` |
| Base URL | `https://api.deepseek.com` |
| 模型名 | `deepseek-v4-flash` |
| max_tokens | `8192` |
| temperature | `0.2` |

保存后先点“测试”，测试成功再设为默认。

## 5. 怎么临时给外部访问

适合临时演示，不需要买服务器。

1. 先在本机启动后端。
2. 到 `frontend` 目录运行：

```powershell
npm run build
npm run preview
```

3. 用 Cloudflare Tunnel 暴露 `http://localhost:4173`：

```powershell
cloudflared tunnel --url http://localhost:4173
```

4. 复制终端里出现的 `https://xxxx.trycloudflare.com` 地址发给别人。

注意：这个地址是临时的，关掉 tunnel 后就失效。长期外网访问建议用 Cloudflare 命名隧道或部署到云服务器。

## 6. 常见问题

| 问题 | 怎么办 |
|---|---|
| 页面打不开 | 先运行 `stop.cmd`，再运行 `start.cmd`。 |
| 刚启动没有数据 | 等 1 到 2 分钟，模拟器会自动产生数据。 |
| AI 一直思考中 | 先点“重新生成”；如果仍不行，到“AI 调用统计”看失败原因。 |
| 通知渠道显示未真实发送 | 正常，这是演示模式，避免真的打扰别人。 |
| 想重置演示数据 | 先停止系统，删除 `backend/data/`，再重新启动。 |

## 7. 演示前检查清单

- `http://localhost:5173` 可以打开。
- “总览大屏”有数据。
- “通知渠道”今日失败为 0，渠道卡片显示“演示成功 · 未真实发送”。
- “系统设置”的默认大模型测试成功。
- “AI 调用统计”能看到调用记录。
