# AIOps Alert · 演示交付清单

> 截至 2026-05-26 当前状态，给比赛演示准备的"功能清单 / 缺口分析 / 需要准备的材料"。

## 一、完整功能清单（已实现 + 计划中）

### 1. 监控对象管理 ✅
- 4 类对象：服务器 / 数据库 / 数据同步作业 / 数据加工作业
- CRUD + 启停 + 类型筛选 + 关键字搜索
- 扩展属性 JSON（IP / 端口 / 作业编码等差异化字段）
- 标签系统

### 2. 通知渠道管理 ✅
- 3 类渠道：企业微信 webhook（真实可用）/ 邮件 SMTP（真实可用）/ 短信（dryRun 演示模式）
- 渠道连通性测试 + 测试结果实时反馈
- CRUD + 启停 + 优先级 + 配置 JSON

### 3. 告警规则管理 ✅
- 多条件 AND/OR 组合
- 13 个对象类型的内置指标字典（CPU / 内存 / 主从延迟 / 慢查询 等）
- 触发策略：连续触发次数 / 观察窗口 / 最小告警间隔 / 重复通知 / 恢复通知
- 4 个告警级别（NOTICE / NORMAL / SERIOUS / CRITICAL）
- 关联对象 + 关联渠道 + 接收人配置

### 4. 告警事件中心 ✅
- 全生命周期：待处理 / 已确认 / 已恢复 / 已关闭
- 处理动作流水（CONFIRM / RECOVER / CLOSE / COMMENT）
- 通知发送流水 + 失败重发
- SSE 实时推送 + 浏览器右上角 toast
- 状态侧栏 + 实时事件流 + 详情抽屉

### 5. Incident 智能归并 ✅
- 同对象 30 分钟窗口内多条告警自动合并为一个 Incident
- 故障组卡片 + 时间轴展示
- top_level 自动取级别最高
- 全部事件恢复时自动关闭 Incident

### 6. 总览大屏 ✅
- 4 张大数字 hero（待处理 / 累计 / 紧急严重 / 活跃 Incident）
- 7 日趋势图（总量 / 待处理 / 已恢复 / 紧急 4 条线）
- 状态 / 级别双圆环图
- 规则命中 Top（带渐变进度条）
- 对象类型分布
- 最近事件 feed
- 实时时钟 + SSE 状态指示

### 7. AI 五大能力 ⭐ 核心卖点

#### 7.1 NL2Rule 自然语言建规则 ✅
- 一句话生成完整告警规则
- AI 自动选择对象 / 指标 / 阈值 / 级别 / 渠道
- 后端二次校验对齐字典
- AI 思考过程展示（reasoning_content）

#### 7.2 AI 告警摘要 ✅
- 异步生成 4 段：发生了什么 / 影响范围 / 可能原因 / 建议动作
- SSE 推送给前端，骨架屏 → 流式渲染
- 思考过程可回看

#### 7.3 智能告警归并 ✅
- 同对象时间窗 30 分钟自动合并
- 时间轴展示故障演进

#### 7.4 阈值智能推荐 ✅
- 基于历史指标 P50/P95/P99 分位数
- 三档推荐：高/中/低敏感度
- 历史数据不足时回落到经验值

#### 7.5 命令面板 Cmd+K ✅
- 自然语言查询当前告警态势
- 4 类意图：list_events / count_events / route / unknown
- 关键词兜底（无 LLM 时也可用）

### 8. 系统设置 ✅
- LLM 模型配置（DeepSeek / OpenAI / Qwen 兼容）
- 测试连通 + 思考过程展示
- 默认模型切换
- 4 个快速预设

### 9. 演示辅助 ✅
- 一键 seed 演示数据（7 对象 + 3 渠道 + 6 规则 + 6 历史事件）
- 一键清空业务数据
- 模拟器故事模式（指标自动爬升 → 自动触发告警）
- 强制触发故事接口

### 10. 实时通信 ✅
- SSE 长连接（20 秒心跳 + 30 分钟超时）
- 前端 60 秒看门狗自动重连
- 4 种事件：connected / event-created / event-updated / ai-summary

### 11. 主题系统 ✅
- 深色 / 浅色 / 跟随系统三态切换
- localStorage 持久化偏好
- 防首屏闪白

### 12. UI 设计语言 (重做中)
- Editorial × Terminal 风格
- Space Grotesk + JetBrains Mono + Noto Sans SC 字体栈
- electric cyan 主色调
- ✅ Dashboard / Events / Incidents 已重做

## 二、距离演示 MVP 的缺口分析

### 🔴 必须修（不修会破坏演示）

#### 1. Rules 等 4 个页面的 UI 重做
- 文件：RulesView / ObjectsView / ChannelsView / SettingsView
- 影响：演示路径会经过这几页，风格不一致评委会扣印象分
- 工作量：4-5 小时
- 优先级：**P0**

#### 2. 浅色主题硬编码颜色 bug
- ChannelsView line 762 `background: #0F172A`
- CommandPalette line 219 渐变写死
- 影响：浅色主题切到这两处时显示破损
- 工作量：5 分钟
- 优先级：**P0**

#### 3. AI 摘要慢的问题
- DeepSeek V4 Pro 思考耗时可能较长，演示默认使用 DeepSeek V4 Flash
- 缓解：seed 时预生成几条事件的 AI 摘要（已有部分），或事件中心默认显示已有摘要的事件在前
- 工作量：1 小时
- 优先级：**P0**

### 🟡 强烈建议补（直接拿分）

#### 4. AI 调用统计页（"AI 工程化"卖点）
- 利用现有 `ai_call_log` 表
- 内容：今日调用次数 / token 消耗 / 成功率 / 场景分布 / 慢调用 Top
- 工作量：半天
- 优先级：**P1**（评委看到 AI 不只是噱头是工程化能力，是差异化点）

#### 5. AI 摘要 / NL2Rule / 命令面板的"流式输出"动画
- 当前是一次性返回，可以改成打字机效果模拟流式（哪怕后端不真流，前端逐字渲染）
- 工作量：3 小时
- 优先级：**P1**（视觉冲击大）

#### 6. 演示数据完善
- 7 天历史事件分布到不同日期，让趋势图有曲线（当前主要集中在今天）
- 给 1-2 条历史事件预生成 AI 摘要
- 工作量：1 小时
- 优先级：**P1**

#### 7. NL2Rule 增强：再加一个"刚才的规则" + "改成更敏感"等迭代式对话
- 工作量：半天
- 优先级：**P2**

### 🟢 锦上添花（有时间做）

#### 8. 每日态势简报（定时任务）
- 8:00 调 LLM 写一段 24 小时简报
- 推送到企微
- 工作量：半天

#### 9. 大屏全屏模式（F11 进入）
- 隐藏侧栏顶栏 + 30s 自动刷新
- 工作量：2 小时

#### 10. 告警关联根因分析
- 基于 Incident 让 LLM 输出"故障演进时间线"
- 工作量：1 天
